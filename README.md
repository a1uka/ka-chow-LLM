# RAG-ассистент по клиническим рекомендациям РОАГ

## Описание проекта

Система представляет собой RAG ассистент для врачей акушеров-гинекологов, работающий на основе клинических рекомендаций РОАГ (Российского общества акушеров-гинекологов).

Основные возможности:

* Поиск релевантной информации в базе клинических рекомендаций.
* Генерация ответов с использованием локальной языковой модели.
* Работа исключительно на основе предоставленного контекста.
* Обработка экстренных обращений через механизм триггеров.
* REST API на базе FastAPI.

---

# Структура проекта

```text
project/
    server.py              # REST API и логика обработки запросов
    model_prompt.py        # Системный промпт и правила безопасности
    chunks.py              # Подготовка текстовых чанков
    metadata.json          # Метаданные документов РОАГ
    faiss_roag_index/      # Векторный индекс FAISS
```

# server.py

Основной сервер приложения.

## Назначение

Файл реализует:

* загрузку эмбеддингов и индекса FAISS;
* поиск релевантных документов;
* взаимодействие с локальной LLM;
* проверку экстренных состояний;
* REST API для подключения.

---

## Конфигурация

```python
FAISS_INDEX_PATH = "faiss_roag_index"
EMBEDDING_MODEL_NAME = "sentence-transformers/paraphrase-multilingual-mpnet-base-v2"
```

### Параметры

| Параметр               | Описание                                        |
| ---------------------- | ----------------------------------------------- |
| `FAISS_INDEX_PATH`     | Путь к индексу FAISS                            |
| `EMBEDDING_MODEL_NAME` | Модель эмбеддингов                              |
| `LLM_BASE_URL`         | URL локальной языковой модели                   |
| `LLM_MODEL`            | Имя модели                                      |
| `LLM_API_KEY`          | Ключ API (для локальной модели не используется) |

---

## Загрузка ресурсов

### `load_resources()`

Загружает:

* модель эмбеддингов HuggingFace;
* индекс FAISS.

При отсутствии индекса генерируется исключение:

```python
FileNotFoundError
```

---

## Проверка экстренных состояний

### `is_emergency(text: str) -> bool`

Проверяет наличие опасных симптомов в пользовательском запросе.

Используются ключевые слова из:

```python
EMERGENCY_TRIGGERS
```

Возвращает:

* `True` — найдено экстренное состояние;
* `False` — состояние не обнаружено.

---

## Вызов языковой модели

### `call_llm(messages, max_retries=2, timeout=15)`

Отправляет запрос локальной языковой модели через OpenAI-compatible API.

### Обрабатываемые ошибки

* `APITimeoutError`
* `APIConnectionError`
* `APIError`
* прочие исключения

При ошибке возвращается сообщение вида:

```text
X <описание ошибки>
```

---

## Поиск ответа

### `ask_assistant(user_query: str, vectorstore) -> str`

Алгоритм:

1. Выполняет поиск в FAISS.
2. Извлекает 4 наиболее релевантных документа.
3. Формирует контекст.
4. Отправляет запрос в LLM.
5. Возвращает ответ модели.

Количество найденных документов:

```python
k=4
```

## API Endpoints

### GET /

Проверка доступности сервиса.

#### Ответ

```json
{
  "status": "ok",
  "message": "Ассистент работает"
}
```

---

### GET /health/llm

Проверка доступности языковой модели.

#### Возможные ответы

Успешно:

```json
{
  "llm_status": "ok",
  "response": "pong"
}
```

Модель недоступна:

```json
{
  "llm_status": "unreachable"
}
```

Таймаут:

```json
{
  "llm_status": "timeout"
}
```

Ошибка:

```json
{
  "llm_status": "error",
  "detail": "..."
}
```

---

### POST /ask

Основной endpoint для получения ответа.

#### Запрос

```json
{
  "query": "Что делать при преэклампсии?"
}
```

#### Ответ

```json
{
  "answer": "...",
  "status": "ok",
  "emergency": false
}
```

---

### Возможные статусы

| Статус      | Описание                        |
| ----------- | ------------------------------- |
| `ok`        | Ответ успешно получен           |
| `emergency` | Обнаружено экстренное состояние |
| `error`     | Ошибка LLM                      |

# model_prompt.py

Файл содержит настройки поведения ассистента.

---

## SYSTEM_PROMPT

Системная инструкция для языковой модели.

Основные ограничения:

* отвечать только по предоставленному контексту;
* не использовать собственные знания;
* отвечать на русском языке;
* указывать источники;
* сообщать об отсутствии информации при отсутствии ответа в рекомендациях.

---

## EMERGENCY_TRIGGERS

Список фраз, сигнализирующих о потенциально опасном состоянии пациента.

Примеры:

```python
"сильное кровотечение"
"судороги"
"потеря сознания"
"не чувствую ребёнка"
"эклампсия"
```

---

## EMERGENCY_RESPONSE

Стандартный ответ при обнаружении экстренной ситуации.

Рекомендует:

* немедленно обратиться за медицинской помощью;
* вызвать скорую помощь;
* связаться с лечащим врачом.

# chunks.py

Подготовка документов для индексирования. Используется просто для обработки текста в векторную БД.

---

## Назначение

Скрипт:

1. Загружает текстовые рекомендации.
2. Создаёт объекты Document.
3. Делит тексты на чанки.
4. Подготавливает данные для построения векторного индекса.

---

## Источник данных

Каталог:

```text
roag_texts/
```

Обрабатываются все файлы:

```text
*.txt
```

---

## Отбор документов

Документы короче 100 символов пропускаются:

```python
if len(content) < 100:
    continue
```

---

## Метаданные

Для каждого документа сохраняется:

```python
metadata={
    "source": txt_file.name
}
```

Это позволяет позже показывать источник ответа.

---

## Параметры разбиения

```python
chunk_size = 1000
chunk_overlap = 100
```

### Значения

| Параметр     | Значение      |
| ------------ | ------------- |
| Размер чанка | 1000 символов |
| Перекрытие   | 100 символов  |

---

## Разделители

Используются следующие уровни разделения:

```python
[
    "\n\n",
    "\n",
    ".",
    " ",
    ""
]
```

---

## Результат

После обработки выводится количество созданных чанков:

```text
Всего чанков: N
```

# metadata.json

Файл содержит метаданные клинических рекомендаций РОАГ.

---

## Назначение

Используется для:

* хранения информации о документах;
* связи PDF и TXT версий;
* учёта размеров текстов;
* последующей навигации по источникам.

---

## Структура объекта

```json
{
  "index": 1,
  "title": "Название рекомендации",
  "google_drive_url": "...",
  "pdf_path": "...",
  "txt_path": "...",
  "text_length": 105643
}
```

---

## Поля

| Поле               | Описание                          |
| ------------------ | --------------------------------- |
| `index`            | Уникальный номер документа        |
| `title`            | Название клинической рекомендации |
| `google_drive_url` | Ссылка на оригинальный PDF        |
| `pdf_path`         | Локальный путь к PDF              |
| `txt_path`         | Локальный путь к текстовой версии |
| `text_length`      | Размер текста в символах          |

# Общий алгоритм работы

```text
Пользователь -> POST /ask -> Проверка emergency триггеров -> Да -> Экстренный ответ
                                            |
                                            ▼
                                           Нет -> Поиск в FAISS -> Получение top4 чанков -> Формирование контекста -> Запрос к локальной LLM -> Возврат ответа пользователю
```
